Hospital Management System

Design and Simulation of a Hospital Management Network Using Cisco Packet Tracer. The healthcare network links different departments including IT, Clinical, Reception, General Ward, Private Ward, and Guest Lobby with routers, switches, and wireless access points. The simulation confirms inter-department communication, server access, and wireless guest access.

Overview

The project makes use of Cisco Packet Tracer where a scalable and secure network of a hospital is simulated. It addresses inter-router communication, departmental segregation, and server access by multiple subnets.

Objectives:

  • Design modular hospital LAN segmented by department
  • Install static IP routing between routers for inter-department communication
  • Deploy centralized DNS, HTTP, FTP servers
  • Configure wireless guest network isolated from internal hospital network
  • Test network performance via ICMP and service-based testing

Configuration

Network Topology:

  • IT Department (192.168.1.0/24)
  • Clinical Area (192.168.2.0/24)
  • Reception Area (192.168.3.0/24)
  • General Ward (192.168.4.0/24)
  • Private Ward (192.168.5.0/24)
  • Guest Lobby (10.0.0.0/24)

Routers connected with serial DCE cables on point-to-point subnets (192.168.6.0/30, 192.168.7.0/30, 192.168.9.0/30).

Server Deployment: DNS, HTTP, and FTP servers hosted under IT and Reception departments, accessible from all subnets.

Wireless Access: DHCP-enabled wireless router issues dynamic IP to guests, restricted from internal access.

Routing: Static routing configured on all routers with next-hop IPs for inter-department communication.

Demonstration

Simulation results show:

  • Effective communication across departments
  • Servers are accessible from all subnets
  • Wireless clients receive DHCP IP and access web services securely
  • Static routing ensures proper packet forwarding

Network performance verified through ICMP ping tests, HTTP/FTP access tests, and DHCP verification on wireless clients.
